# Agent Tools

The BrickForge agent runtime discovers and loads tools dynamically at startup based on `config.json`. No hardcoded tool list - everything is config-driven.

## Tool types

### UC Functions (auto-discovered)

Unity Catalog SQL functions registered in `tools.functions[]`. The agent calls them via `SELECT * FROM catalog.schema.func(params)`.

- **Discovery**: `discover_uc_function_tools()` in `brickforge/tools/tool_factory.py` reads function names from config, fetches UC metadata via `w.functions.get()`, and creates `sql_read` tools.
- **Config**: `tools.functions` array in `config.json`
- **Registration**: Auto-appended by `generate_routines.py` after provisioning

```json
{
  "tools": {
    "functions": [
      "catalog.schema.get_top_customers",
      "catalog.schema.flight_delay_stats"
    ]
  }
}
```

### Genie (MCP)

Natural-language SQL via Databricks Genie spaces. The agent asks Genie a question, Genie writes SQL, queries tables, returns results.

- **Discovery**: Genie space IDs from `tools.genie_spaces[]`
- **Config**: Array of Genie space IDs
- **Runtime**: MCP-based tool call. SQL generated by Genie is captured and logged via `brickforge/agent/genie_capture.py`
- **Grant**: Object type `genie`, permission `CAN_RUN`

### Knowledge Assistants

RAG over documents via Databricks Knowledge Assistant endpoints.

- **Discovery**: `brickforge/tools/ka_factory.py` reads `tools.ka.{SLUG}` entries from config
- **Config**: `tools.ka.{SLUG}.endpoint` + `tools.ka.{SLUG}.enabled`
- **Double gate**: Tool only loads if BOTH `bricks.KA.enabled` AND `tools.ka.{SLUG}.enabled` are true
- **Grant**: Serving endpoint with `CAN_QUERY`

!!! warning
    Brick toggles (`bricks.KA.enabled`) are not yet enforced at agent runtime (see [Known Issues](reference/known-issues.md#44-brick-toggles-not-enforced-at-runtime)). Tools load regardless of toggle state.

### Vector Search

Semantic retrieval via Databricks Vector Search indexes.

- **Discovery**: `tools.vector_search.index` + `tools.vector_search.endpoint` in config
- **Config**: Index name and endpoint name

### External APIs

REST API tools via UC Connections or direct HTTP.

- **Discovery**: `brickforge/tools/api_factory.py` reads `tools.api.{SLUG}` entries
- **Config fields**: `conn` (UC connection name), `url`, `method`, `path`, `desc`, `params`, `header`, `enabled`
- **UC Connection**: Managed credentials - the API key is stored in Unity Catalog, not in config
- **Direct HTTP**: URL + auth header in config

### MCP Servers

External Model Context Protocol servers.

- **Discovery**: `tools.mcp.{SLUG}` entries in config
- **Config fields**: `url`, `header`, `enabled`
- **Runtime**: Each MCP server exposes tools the agent can call

### A2A Agents

Agent-to-agent connections via Google's A2A protocol.

- **Discovery**: `brickforge/tools/a2a_factory.py` reads `tools.a2a.{SLUG}` entries
- **Config fields**: `url`, `header`, `enabled`
- **Runtime**: The remote agent becomes a callable tool

### Charts

Inline chart generation in chat responses.

- **Source**: `brickforge/tools/generate_chart.py`
- **Feature gate**: `features.CHART.enabled`
- **Runtime**: Agent generates chart specs, rendered inline in the chat UI

### Memory

Per-user long-term memory stored in Lakebase.

- **Source**: `brickforge/agent/memory_tools.py`
- **Feature gate**: `features.MEMORY.enabled`
- **Storage**: `AsyncDatabricksStore` backed by Lakebase (Postgres-compatible)
- **Scope**: Per-user - each user gets their own memory namespace

### Current Time

Simple utility tool that returns the current timestamp.

- **Source**: `brickforge/tools/get_current_time.py`
- **Always loaded**: No feature gate

## Tool loading flow

At agent startup (`brickforge/agent/agent.py`):

1. Read `config.json` via `flatten()` into env vars
2. `tool_factory.py` iterates over config sections:
    - `tools.functions[]` -> UC function tools
    - `tools.genie_spaces[]` -> Genie MCP tools
    - `tools.ka.{SLUG}` -> KA tools (if double-gated)
    - `tools.mcp.{SLUG}` -> MCP tools
    - `tools.api.{SLUG}` -> API tools
    - `tools.a2a.{SLUG}` -> A2A tools
3. Feature-gated tools checked against `features.{NAME}.enabled`
4. All tools passed to LangGraph agent as LangChain tools

## Enable/disable

Multi-instance tools (KA, MCP, API, A2A) have an `enabled` boolean per entry. Disabled entries are preserved in config but omitted from `flatten()` output, so the agent never sees them.

```json
{
  "tools": {
    "mcp": {
      "weather": { "url": "https://mcp.example.com", "enabled": true },
      "disabled-tool": { "url": "https://old.example.com", "enabled": false }
    }
  }
}
```
