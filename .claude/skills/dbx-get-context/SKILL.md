# Get Project Context

Scan the codebase and docs to rapidly understand what this project does, how it's structured, and how to work with it. Produces (or reads) a persistent context summary at `docs/context-for-claude-code.md`.

## Steps

### 1. Check for existing context file

```bash
test -f docs/context-for-claude-code.md && echo "EXISTS" || echo "NOT_FOUND"
```

- **EXISTS** -> Read `docs/context-for-claude-code.md` and present its contents to the user. Then skip to step 4.
- **NOT_FOUND** -> Continue to step 2.

### 2. Scan the project

Read the following in parallel to build a mental model:

**Docs (read in full):**
- `docs/agent-forge_overview.md`
- `docs/agent-forge_guide.md`
- `docs/Build & setup flow.md`
- `docs/known-issues.md`

**Code structure (read first ~80 lines of each):**
- `agent/agent.py` — main agent logic
- `agent/start_server.py` — server entrypoint
- `agent/utils.py` — utilities
- `app.yaml` — Databricks Apps config
- `databricks.yml` — DAB bundle config
- `pyproject.toml` — Python project metadata
- `requirements.txt` — dependencies

**Tool files:**
- Glob `tools/*.py` and read the first ~30 lines of each to capture docstrings and function signatures.

**Config files:**
- Glob `conf/**/*.yml` and read the first ~20 lines of each.

**Scripts:**
- Glob `scripts/**/*.{py,sh}` — list them and read the first ~15 lines of each to capture purpose comments.

**Chat UI (end-user frontend):**
- Glob `app/**/*.{js,jsx,ts,tsx}` — list files. Read main entry points first ~50 lines.
- This is the deployed React + Node.js chat app that end users interact with on Databricks Apps.

**Visual (local dev dashboard — NOT the chat UI):**
- Glob `visual/**/*.{js,jsx,ts,tsx}` — list files. Read `visual/backend/index.js` and `visual/frontend/src/App.tsx` first ~50 lines.
- This is a local-only React Flow dashboard for architecture visualization, setup wizard, and env editing.

**Prompt templates:**
- Glob `conf/prompt/**/*` and read each file.

### 3. Write the context file

Create `docs/context-for-claude-code.md` with the following structure. Be concise but complete -- this file is meant to be quickly loaded in future conversations to avoid re-scanning.

```markdown

> Auto-generated by /dbx-get-context. Update by deleting this file and re-running.

## What this project is

<2-3 sentence summary of purpose and target user>

## Architecture

<Describe the layers: frontend, backend/API proxy, agent server, tools, data layer, model serving>

## Key directories

| Directory | Purpose |
|-----------|---------|
| `agent/`  | ... |
| `tools/`  | ... |
| `conf/` | ... |
| `conf/prompt/` | ... |
| `scripts/`| ... |
| `visual/` | Local dev dashboard (React Flow) — architecture graph, setup wizard, env editor. NOT the chat UI |
| `app/` | End-user chat UI (React + Node.js proxy) deployed on Databricks Apps |
| `docs/`   | ... |

<List each tool file with a one-line description of what it does>
```

### 4. Present the context

Print a concise structured summary to the user covering:
- What the project is
- Architecture overview
- Key directories and their roles
- Available tools
- How to deploy and evaluate

Format with headers and tables. Keep it scannable.
